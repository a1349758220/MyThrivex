# 2026-05-08 首页暗色流星效果

## 背景

本次改动为博客首页主图区域增加暗色模式下的 CSS 流星效果。流星只在首页 `Slide` 背景图范围内显示，并依赖父容器 `overflow-hidden` 在越界时裁切消失。

## 变更范围

### 修改文件

- `src/app/page.tsx`
- `src/components/Slide/index.tsx`
- `src/styles/index.scss`

### 新增内容

- `Slide` 组件新增 `showMeteors?: boolean` 开关。
- 新增 `MeteorShower` 内部组件。
- 新增流星随机参数生成逻辑。
- 新增 `.home-meteor-layer`、`.home-meteor` 与 `@keyframes home-meteor-flight` 样式。
- 首页 `<Slide>` 使用 `<Slide showMeteors>` 开启流星效果。

### 删除内容

- 首页移除了旧的 `Typed` 打字机组件引入与使用。
- 首页移除了 `getWebConfigDataAPI`、`Theme` 相关获取旧首页背景图配置的代码。
- 首页从旧 `Starry` 组件切换为 `StarrySky` 组件。

### 流星随机生成逻辑

位置：`src/components/Slide/index.tsx`

```tsx
const METEOR_COLORS = ['#ffffff', '#bfe4ff', '#f7d7ff', '#ffd6c9', '#d8fff4'];

const randomBetween = (min: number, max: number) => Math.random() * (max - min) + min;

const createMeteor = (id: number): Meteor => {
  const isLeftToRight = Math.random() > 0.5;
  const distance = randomBetween(150, 190);

  return {
    id,
    style: {
      '--x': `${randomBetween(isLeftToRight ? -28 : 58, isLeftToRight ? 38 : 128).toFixed(2)}%`,
      '--y': `${randomBetween(-90, 60).toFixed(2)}px`,
      '--angle': `${randomBetween(isLeftToRight ? 35 : 105, isLeftToRight ? 75 : 145).toFixed(2)}deg`,
      '--distance': `${distance.toFixed(2)}vh`,
      '--end-distance': `calc(${distance.toFixed(2)}vh - 100px)`,
      '--delay': `${randomBetween(0, 10).toFixed(2)}s`,
      '--length': `${randomBetween(150, 260).toFixed(2)}px`,
      '--height': `${randomBetween(2, 3.2).toFixed(2)}px`,
      '--color': METEOR_COLORS[getRandom(0, METEOR_COLORS.length - 1)],
    },
  };
};
```

### MeteorShower 组件

位置：`src/components/Slide/index.tsx`

```tsx
function MeteorShower() {
  const meteorIdRef = useRef(0);
  const [meteors, setMeteors] = useState<Meteor[]>([]);

  useEffect(() => {
    setMeteors(Array.from({ length: 2 }, () => createMeteor(meteorIdRef.current++)));
  }, []);

  if (!meteors.length) return null;

  return (
    <div className="home-meteor-layer" aria-hidden="true">
      {meteors.map((style) => (
        <span
          key={style.id}
          className="home-meteor"
          style={style.style}
          onAnimationIteration={() => {
            setMeteors((prev) => prev.map((item) => (item.id === style.id ? createMeteor(meteorIdRef.current++) : item)));
          }}
        />
      ))}
    </div>
  );
}
```

## 后续调整

### 1. 显示范围调整

- 初始实现将流星层放入 `Slide` 背景容器内部。
- 依赖 `Slide` 的 `overflow-hidden` 裁切越界内容，确保流星只在首页主图背景图范围内显示。
- `.home-meteor-layer` 通过 `html.dark .home-meteor-layer { display: block; }` 限制为暗色模式显示。

### 2. 层级调整

- 由于 `StarrySky` 的 canvas 会覆盖背景区域，流星层改为在 `{children}` 后渲染。
- `.home-meteor-layer` 设置 `z-index: 15`，保证流星显示在星空 canvas 之上。

### 3. 数量与冷却调整

- 流星数量从多枚逐步减少为 `2` 枚。
- 动画周期固定为 `10s`。
- 每个周期内最多渲染两枚流星，满足“10 秒内最多出现 2 枚”的节奏要求。

### 4. 速度调整

- 早期版本使用随机速度。
- 当前版本统一使用 `animation: home-meteor-flight 10s linear var(--delay) infinite both;`。
- 可见飞行阶段集中在动画前 `64%`，整体可见时间大于 3 秒。

### 5. 方向调整

- 早期版本偏向单一方向。
- 当前版本使用 `isLeftToRight` 随机决定方向：
  - 左上到右下：角度范围约 `35deg ~ 75deg`
  - 右上到左下：角度范围约 `105deg ~ 145deg`

### 6. 出现位置调整

- 流星出现位置改为页面顶部附近。
- `--y` 使用 `-90px ~ 60px`，避免从页面中部突然出现。
- `--x` 根据方向生成：
  - 左往右：`-28% ~ 38%`
  - 右往左：`58% ~ 128%`

### 7. 消失位置调整

- 消失距离从原始 `--distance` 调整为 `--end-distance`。
- 当前 `--end-distance` 计算方式为：`calc(distance vh - 100px)`。
- 目的是让流星比原始路径终点提前约 `100px` 消失。

### 8. DOM 属性残留处理

- 早期版本在 `onAnimationIteration` 中直接调用 `event.currentTarget.style.setProperty(...)` 修改 CSS 变量。
- 后续改为通过 React state 重新生成流星对象，并使用新的 `key` 重新挂载节点。
- 这样可以避免上一轮动画的 CSS 自定义属性残留到下一轮。

## 当前行为摘要

- 仅暗色模式显示。
- 首页 `Slide` 背景图区域内显示。
- 2 枚流星循环出现。
- 每 10 秒最多出现 2 枚。
- 方向随机，支持左往右与右往左。
- 位置、角度、颜色、长度、高度、延迟每轮重新随机。
- 越出 `Slide` 容器后由 `overflow-hidden` 裁切。

## 备注

当前实现的消失位置是沿流星飞行路径提前 `100px`，不是严格按页面垂直方向固定在背景图底部上方 `100px`。如果后续需要精确的垂直终点，应将动画从单轴 `translateX` 改为二维 `translate3d(x, y, 0)`，并将 `--end-y` 明确计算为 `100vh - 100px - 起点Y`。
